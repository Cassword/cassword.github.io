    CMP417 - Engineering Resilient Systems

    Overall Grade: A

    Part 1 - Software Security

    Grade: A

    The entire premise of this module was that we were an IT manager of a small energy company. We were asked to pick from a given list of identified software systems that could be exploited, we were asked to choose one, then find similar systems that had been exploited on MITRE's CVE database. We then had to identify a class of security faults that might lead to a vulnerability on this system. We were then asked to recommend a software development practice that could be used to prevent this vulnerability in the context of the company (practicality, effectiveness, comparison with other practices) and specifically how it might have prevented an existing instance of exploitation.

    I chose static analysis as my method and employed both Semgrep and Snyk to do it.

    Part 2 - Machine Learning

    Grade: B+

    This task asked us to design a machine learning classifier to categorize either network packet data or system memory dump files according to their attack category. This involved discussing two machine learning algorithms suitable for this problem including their strengths, weaknesses and limitations. We had to outline all stages of building a machine learning model including data ingestion/preprocessing, modelling, analysis and communication of results. Then, how we would test our proposed model using suitable evaluation metrics.

    I chose an implementation employing Random Forest as the algorithm.

    Part 3 - Human-Centred Security

    Grade: A

    This unit asked us to review the literature on how phishing attacks & authentication design can compromise security then recommend a set of mitigations and mechanisms linked to the employees of the company. Recommendations then had to be made as to an authentication mechanism to secure internal applications considering a balance of security and usability, and methods that could improve resillience to phishing.
